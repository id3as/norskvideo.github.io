Build visually. Deploy instantly.

Design your live workflow using Norsk Studio's drag-and-drop interface. Connect sources, add encoding and packaging, insert graphics and subtitles, set up source switching and monitoring — all without writing code.

Studio supports RTMP, SRT, WebRTC, SDI/HDMI via DeckLink, and more. Add DRM, build ABR ladders, record live-to-VOD, deliver to multiple CDNs — all from the same canvas.

Save any workflow as a reusable template for future productions.

Norsk Studio — Live Event Workflow

Press play to go live.

Run on your own hardware, in the cloud, or both. Norsk runs natively on AWS, Google Cloud Platform, and Akamai Cloud Computing, and works with Docker and Kubernetes.

Every component includes built-in runtime controls — switch sources, update graphics, and adjust settings during a live event. The Norsk Visualizer lets you inspect any workflow in real time, with native support for OpenTelemetry and Fluent Bit.

When you need scale, Norsk Manager provides fault-tolerant infrastructure that keeps your service running even if individual servers or cloud regions go down.

Integrate with everything.

Many Studio components expose an OpenAPI-defined interface — trigger source switches, update overlays, or start recordings with simple HTTP requests. A built-in Swagger interface makes testing easy.

For teams that need to go further, the Norsk Engine SDK provides a low-code TypeScript API for building fully custom media applications on top of the same broadcast-grade engine that powers Studio. Build custom components, or create entirely bespoke streaming services — in days, not months.

Built for reliability. Engineered for the moments that can't fail.

Live video doesn't get a second take. Norsk is engineered for resilience at every level: redundant source acquisition over multiple routes, automatic failover across servers and cloud regions, and infrastructure that keeps running even when individual components fail.

With Norsk Manager, you can deploy workflows where they make the most sense — encode at the venue for lowest latency, produce in the cloud for flexibility, or run a hybrid setup that adapts to each event's needs. Your service stays up, and your viewers never notice the complexity behind it.
