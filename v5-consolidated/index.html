How It Works

Build. Run. Connect.

Three steps from first workflow to production deployment.

Step 01 — Build

Design workflows visually, no code required

Drag sources, encoders, packagers, and outputs onto the canvas. Connect them with a click. Add graphics overlays, set up source switching, configure ABR ladders — all from a single visual interface. Save any workflow as a reusable template.

RTMP SRT SDI WebRTC HLS DASH
studio — canvas

Step 02 — Run

Deploy anywhere with a single click

Press play to deploy on your own hardware, in the cloud, or both. Norsk runs natively on AWS, GCP, and Akamai. Switch sources and update graphics live without interrupting the stream — all from the same UI.

AWS GCP Akamai Docker Kubernetes Bare metal

Step 03 — Connect

Control everything via API

Trigger source switches, update overlays, or start recordings with a single HTTP request. Go deeper with the Norsk Engine SDK to build fully custom media applications — in TypeScript, in days rather than months.

OpenAPI TypeScript SDK Swagger UI Webhooks

// Switch to backup camera
await studio.switchSource({
