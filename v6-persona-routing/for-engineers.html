What Engineers Build

Common workflow scenarios in Norsk Studio

The kinds of pipelines workflow engineers build and operate with Studio — with room for your team's story as we grow together.

Broadcast Engineer

Replacing a multi-tool stack with a single visual platform

A workflow engineer was maintaining a pipeline built from multiple discrete tools — each with its own configuration format, monitoring interface, and failure mode. Debugging cross-tool issues was time-consuming and opaque.

A single Norsk Studio workflow replaced the stitched-together stack. One visual interface for configuration and monitoring; per-node telemetry made it immediately clear where problems were occurring, rather than narrowing down between tools.

Talk to us about your stack →
Workflow Engineer

Multi-protocol pipeline with per-stream monitoring

An engineer needed to build an ingest pipeline accepting SRT and RTMP, transcoding to multiple profiles, and delivering to HLS, DASH, and a WebRTC monitoring endpoint — with per-stream visibility for operations throughout.

Built in Studio's visual builder with minimal coding — ingest and delivery configuration handled through the interface. The auto-generated OpenAPI spec let operations integrate workflow control into existing tooling. Per-node bitrate, latency, and error telemetry available in the Studio dashboard from first run.

Talk to us about your requirements →
Operations Engineer

Wiring live workflow control into an existing orchestration system

An operations team needed to control Norsk workflows programmatically from an existing orchestration system — triggering source switches, adjusting output targets, monitoring stream health — without building a separate control plane.

Norsk's auto-generated OpenAPI spec gave the orchestration system a typed REST interface to every workflow. The team wired Norsk into their existing control plane in a day. No custom integration layer required.

Capabilities

Built for the engineer, not just the operator

Studio's visual surface sits on top of a fully programmable engine. You choose how deep to go.

Builder
Visual Workflow Designer

Drag, connect, and configure pipeline nodes with minimal coding. Every workflow is also represented as a versioned JSON graph you can store in Git.

API
Auto-Generated OpenAPI

Every workflow you build automatically exposes a typed REST API. Control workflows programmatically from your orchestration layer.

Monitoring
Per-Node Telemetry

Frame-level metrics, bitrate graphs, and error events for every node in your pipeline — in real time, in the browser.

Protocols
Multi-Protocol Ingest & Egress

SRT, RTMP, WebRTC, MoQ, CMAF, HLS, DASH, SDI gateway, MXL, and TAMS. Sub-second MoQ delivery at CDN scale; sub-millisecond MXL intra-server exchange; TAMS live-to-VOD without pipeline restarts.

Extensibility
Norsk Engine SDK

TypeScript-first SDK for building custom media processing nodes. When you need behavior Studio's visual builder doesn't cover, the SDK has you.

Deployment
On-Prem, Cloud, or Hybrid

Run Norsk where your media already lives. No forced cloud migration; no vendor lock-in on the infrastructure layer.
